ThreadWeaver
Consent-first conversational memory for Goose sessions

ThreadWeaver brings prior thinking into current build sessions with policy-scoped retrieval and project-aware context continuity.

GitHub Repository @threadweaver/cli @threadweaver/sdk AAIF ecosystem context

Usage

Goose-First Workflow

Prompt naturally in session

I remember thinking in ChatGPT about X. Surface my thought and connect it to what we're building on project Y.

ThreadWeaver response model

Intent is inferred, retrieval is policy-governed, and synthesis stays scoped to approved project context.

Install

Local Setup

git clone https://github.com/dakotafabro/threadweaver.git cd threadweaver npm install npm run -w @threadweaver/cli dev -- init

Design

Core Decisions

Local-first and consent-first

Memory remains local by default and retrieval is deny-by-default until project access is explicitly granted.

Deterministic policy enforcement

Policy checks are strict regardless of parser path, with traceable decisions for auditability and trust.
